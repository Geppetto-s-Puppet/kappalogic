# n個並べた加算器へのゲージ理論的アプローチ — AND/ORの非対称性の起源

`gauge.py`(単一の検出器 `k(x;ξ)=tanh(x/ξ)` が持つ Aff(1,R) 対称性・
双曲計量のキリングベクトル場)を、「n個並べた検出器」「AND_n」
「n bitのリップルキャリー加算器」に拡張して調べた。付属の
`gauge_adder_chain.py`を実行すればここに書いた結果は全部再現できる。

## 1. n個の独立な検出器: 対称性群は aff(1,R) の直和

各サイトiに独立な`(x_i, ξ_i)`を割り当て、gauge.pyと同じ生成子
(並進`T_i=∂/∂x_i`、ディレーション`D_i=x_i∂/∂x_i + ξ_i∂/∂ξ_i`)を置くと、
sympyでの交換子計算により

```
[D_i, T_i] = -T_i        (同じサイト、gauge.pyの単一サイト結果と一致)
[D_i, T_j] = 0  (i≠j)    (異なるサイトは完全に無関係)
[D_i, D_j] = 0  (i≠j)
[T_i, T_j] = 0  (i≠j)
```

が厳密に成り立つ(n=3で確認)。つまり「n個の独立な検出器」の対称性は
単に**aff(1,R)のn個の直和**であり、これ自体は驚くことではない
(独立な変数を並べただけなので当然)。対応する幾何は双曲上半平面
`H²`のn個の直積`(H²)ⁿ`で、`T_i`・`D_i`はこの直積計量
`ds²=Σ(dx_i²+dξ_i²)/ξ_i²`のキリングベクトル場になっている(sympyで確認済み)。
ここまでは「当然のこと」を確認しただけ。

## 2. 本題: AND(a,b;ξ)は「重み(1,1,2)」の拡張ディレーション対称性を持つが、OR/XORは持たない

面白かったのはここから。AND・OR自体は「独立な検出器」ではなく、
**2つの引数を掛け合わせてから同じξに通す**という構造をしている
(`AND(a,b;ξ)=reg(a·b;ξ)`)。この掛け算のせいで、gauge.pyの単純な
`(x,ξ)→(λx,λξ)`という対称性がそのままの形では通用しない。

正しい変換則を探すと:

```
AND(λa, λb; λ²ξ) = AND(a, b; ξ)     を厳密に満たす(証明: sympyで代数的に0を確認、
                                       乱数200点でも最大差 1.7e-16 = 浮動小数点誤差のみ)
```

つまり**ξは(a,b)それぞれの重み1の合計=重み2でスケールしないと不変にならない**。
これは単なる次元解析だが、`ab`が掛け算(次数2)である以上、`ξ`も次数2で
追従する必要がある、というだけの話ではある。しかし**OR/XORは、この
(1,1,2)の重みでも、素朴な(1,1,1)の重みでも、どちらの変換でも不変には
ならない**(乱数200点で最大差0.53、0.96——決して小さい"近似的なズレ"
ではなく、対称性が本質的に破れている):

```
OR (λa,λb;λ²ξ) - OR(a,b;ξ)  の最大差 ≈ 0.53   (不変ではない)
XOR(λa,λb;λ²ξ) - XOR(a,b;ξ) の最大差 ≈ 0.96   (不変ではない)
```

**これは`theory.py`の命題5(「ANDは`a·b/ξ`という1変数に還元できる
(部分ディレーション不変)が、ORはできない」)を、gauge.pyのLie群の言葉で
独立に"再導出"したものになっている**。命題5は数値実験で見つかった経験則
だったが、今回の結果は「なぜそうなるか」への一つの説明を与える:
AND(a,b;ξ)は掛け算`a·b`という1つの結合済みの量だけに依存する構造をして
いるので、その量にξを揃えて追従させれば必ず不変な"軌道"を作れる
(=対称性の生成子が具体的に書ける)。OR/XORは`NOT(NOT(a)·NOT(b))`という
形で、ξがa,bそれぞれに対して"2段階"に効いてくる(NOTの内側で1回、
外側でもう1回)ため、どの重みを選んでも整合しない。gauge.py・theory.py
という別々のモジュールで発見されていた2つの事実(Aff(1,R)対称性、
AND/ORの非対称性)が、実は同じ現象の別の顔だった、というのが今回の
一番の収穫だと思う。

## 3. AND_nは重みnで一般化され、対応する幾何が構成できる

2引数のANDが重み2で不変だったので、n引数のAND_nならどうなるか調べた。

```
AND_n(λa_1,...,λa_n; λⁿξ) = AND_n(a_1,...,a_n; ξ)
```

がn=2,3,4,5全部で厳密に成り立つ(sympy的にも数値的にも最大差は浮動小数点誤差
のみ)。つまりAND_nは「重み(1,...,1,n)」の拡張ディレーション対称性を持つ。

この対称性に対応する幾何も実際に作れた。座標`(x_1,...,x_n,ξ)`
(ξは共有、AND_nの実際のAPIと同じ)の上に計量

```
ds² = ξ^(-2/n) · Σ dx_i²  +  ξ^(-2) dξ²
```

を置くと、並進`T_i=∂/∂x_i`と対角ディレーション
`D_n = Σx_i∂/∂x_i + nξ∂/∂ξ`が、この計量のキリングベクトル場になる
ことをsympyで確認した(n=1のときは`ds²=(dx²+dξ²)/ξ²`というgauge.pyの
双曲計量そのものに厳密に一致する — 今回の構成はgauge.pyの一般化に
なっている)。

## 4. これまでの実験結果への接続(仮説、証明ではない)

ここからは推測を含む。これまでの実験(前回のレポート参照)で見つけた
2つの現象——

- (a) 加算器のビット幅nを増やしても「安全なξの上限」が縮まらない
- (b) 加算器へのgradient-based逆算は、ビット幅nを増やすと急激に失敗する
      (単一AND単体の逆算は100%成功するのに)

は、今回の対称性の結果と整合的に説明できるかもしれない。**AND(_n)には
「異なる引数の組を1つの不変量に押し込める」拡張ディレーション対称性が
あるが、加算器はXORを毎段通すため、この対称性を(1個のANDゲート単体を
除いて)全く継承していない**。つまり:

- (a)について: 「ビット幅を増やしても安全域が縮まらない」のは、命題9/10
  のような「複数の生の実数値がスケールを揃えて連鎖する」状況とは違い、
  そもそも各ビット位置の安全性を結びつける連続的な対称性が(XORのせいで)
  存在しないから、各段が独立な局所的性質になっている、と解釈できる。
- (b)について: ANDだけの回路なら、拡張ディレーション対称性に沿って
  なめらかに"滑って"最適解に辿り着けるが、XORが混ざった途端にこの
  滑り台が消えるので、勾配降下は各段の飽和(tanh)を个别に突破しないと
  いけなくなり、段数が増えるほど指数的に難しくなる、と考えると
  4節の実測(n=1:19/20 -> n=5:0/20)と定性的に整合する。

**ただし、これは「もっともらしい物語」であって証明ではない。** 実際に
「対称性の有無」と「勾配降下の成功率」の関係を定量的に結びつける計算は
していないし、他の説明(単に非線形合成が深くなるほど一般に最適化が
難しくなるという、対称性とは無関係などこにでもある事実)を排除できても
いない。ここは今後の宿題として正直に書いておく。

## 誇張しないための注記

- AND(_n)の拡張ディレーション対称性そのものは、突き詰めれば「掛け算の
  次数を数える」という初等的な次元解析に過ぎない。新しい数学ではない。
- 「重み付き半空間」計量`ds²=ξ^(-2/n)Σdx_i²+ξ^(-2)dξ²`がキリング
  ベクトル場を持つことは確認したが、この空間が微分幾何の文献にある
  何らかの標準的な空間(Heintze群/負曲率等質空間のようなものに似ている
  印象を受けたが)と一致するかどうかは確認していない。曲率や測地線の
  完全な解析はしていない。
- 4節の「XORが対称性を壊すので勾配降下が難しくなる」という説明は
  仮説であり、他の(対称性と無関係な)説明を排除できていない。
- OR/XORが「不変にならない」ことは複数の重みで確認したが、
  **他のどんな重み付け・変換群であればOR/XORが不変になるか**は
  探索していない(存在しないと証明したわけではなく、単純な候補が
  外れただけ)。
